Distance boundaries from BFS tree on undirected graphs

Introduction​

As we have talked on the seminar, if we construct from some vertex uuu BFS tree on an undirected graph, we can obtain:

  • lower bound of length of the shortest path between 2 vertices from the height difference
  • upper bound of length of the shortest path between 2 vertices from the path through the root

Lower bound​

Let's consider pair of vertices eee and hhh. For them we can safely lay, from the BFS tree, following properties:

  • lower bound: 222
  • upper bound: 444

By having a look at the graph we started from, we can see that we have a path ‹e,j,he, j, he,j,h› that has a length 2. Apart from that we can also notice there is another path from eee to hhh and that is ‹e,a,c,i,d,he, a, c, i, d, he,a,c,i,d,h›. And that path has a length of 555. Doesn't this break our statements at the beginning? (I'm leaving that as an exercise ;))

Proof by contradiction​

Let's keep the same graph, but break the lower bound, i.e. I have gotten a lower bound 222, but “there must be a shorter path”! ;)

Now the more important question, is there a shorter path in that graph? The answer is no, there's no shorter path than the one with length 222. So what can we do about it? We'll add an edge to have a shorter path. Now we have gotten a lower bound of 222, which means the only shorter path we can construct has 111 edge and that is ‹e,he, he,h› (no intermediary vertices).
